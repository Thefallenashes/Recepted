================================================================================
INSTRUCCIONES Y DOCUMENTACIÓN: upload.php
Ruta: src/pages/scripts/upload.php
================================================================================

PROPÓSITO
---------
Permite a un usuario autenticado subir archivos Excel/CSV al servidor.
Valida el archivo, lo guarda en disco y registra la entrada en la base de datos.


DEPENDENCIAS
------------
- src/pages/scripts/script_bootstrap.php   → require_script_user()
- src/pages/includes/sticky_menu.php       → render_sticky_menu()
- src/utils/db.php                         → getPDO()
- src/css/index.css                        → estilos de la vista
- src/js/sticky-menu-toggle.js             → comportamiento del menú


CONFIGURACIÓN
-------------
- Tamaño máximo de archivo : 10 MB (10 * 1024 * 1024 bytes)
- Directorio de destino    : src/uploads/
- Extensiones permitidas   : csv, xls, xlsx, xlsm, xlsb, xltx

Tipos MIME aceptados por extensión:
  csv  → text/csv, text/plain, application/csv, application/vnd.ms-excel
  xls  → application/vnd.ms-excel, application/octet-stream
  xlsx → application/vnd.openxmlformats-officedocument.spreadsheetml.sheet,
         application/zip, application/octet-stream
  xlsm → application/vnd.ms-excel.sheet.macroenabled.12,
         application/vnd.openxmlformats-officedocument.spreadsheetml.sheet,
         application/zip, application/octet-stream
  xlsb → application/vnd.ms-excel.sheet.binary.macroenabled.12,
         application/octet-stream
  xltx → application/vnd.openxmlformats-officedocument.spreadsheetml.template,
         application/zip, application/octet-stream


FLUJO DE EJECUCIÓN
------------------

1. AUTENTICACIÓN
   - Llama a require_script_user('redirect', '../login.php').
   - Si no hay sesión activa, redirige al login.
   - Si la sesión es válida, devuelve el $userId.

2. PROCESAMIENTO (solo en petición POST)

   2.1 Comprobación de presencia del archivo
       - Si $_FILES['file'] no existe → error.

   2.2 Validación del error PHP
       - Comprueba $_FILES['file']['error'] === UPLOAD_ERR_OK.
       - Errores mapeados a mensajes legibles:
           UPLOAD_ERR_INI_SIZE   → "El archivo supera el límite del servidor."
           UPLOAD_ERR_FORM_SIZE  → "El archivo supera el tamaño máximo permitido."
           UPLOAD_ERR_PARTIAL    → "El archivo se subió de forma incompleta."
           UPLOAD_ERR_NO_FILE    → "No se ha seleccionado ningún archivo."
           UPLOAD_ERR_NO_TMP_DIR → "Falta la carpeta temporal del servidor."
           UPLOAD_ERR_CANT_WRITE → "El servidor no pudo escribir el archivo."
           UPLOAD_ERR_EXTENSION  → "Una extensión de PHP detuvo la subida."

   2.3 Validación de tamaño
       - Si $file['size'] > 10 MB → error.

   2.4 Validación de origen real
       - is_uploaded_file($file['tmp_name']) debe ser true.
       - Previene ataques de path traversal en la subida.

   2.5 Validación de extensión
       - Se extrae con pathinfo() y se sanea con preg_replace (solo [a-zA-Z0-9]).
       - Debe estar en la lista de extensiones permitidas.

   2.6 Validación de tipo MIME
       - Se detecta el MIME real con finfo (FILEINFO_MIME_TYPE) si está disponible.
       - Si finfo no está disponible, se usa el MIME declarado por el cliente como fallback.
       - Se comprueba que el MIME efectivo esté en la lista permitida para esa extensión.
       - Si el MIME detectado no coincide con el declarado → error.

   2.7 Generación del nombre de archivo
       - Formato: {userId}_{timestamp}_{12 hex aleatorios}.{ext}
       - Ejemplo: 42_1712500800_a3f9c1e07b2d.xlsx
       - El sufijo aleatorio se genera con random_bytes(6); fallback a mt_rand + microtime.

   2.8 Guardado en disco
       - Se crea el directorio src/uploads/ si no existe (mkdir 0755 recursivo).
       - Se verifica que el directorio tenga permisos de escritura.
       - Se mueve el archivo con move_uploaded_file().

   2.9 Registro en base de datos
       - Tabla: uploads
       - Columnas insertadas:
           user_id  → ID del usuario en sesión
           filename → basename del nombre original del archivo
           filepath → 'uploads/{newName}' (ruta relativa)
           mime     → MIME efectivo (finfo > cliente > 'application/octet-stream')
           size     → tamaño en bytes
       - Si PDOException → se hace error_log, se borra el archivo con unlink y se muestra error.

3. RENDERIZADO HTML
   - Se muestra siempre (GET y POST).
   - Si hay $mensaje, se renderiza en un <div class="mensaje {tipo}"> escapado con htmlspecialchars.
   - El formulario usa enctype="multipart/form-data" y el atributo accept filtra por extensión en el navegador.
   - Enlace a mis_uploads.php para ver los archivos subidos.


TABLA DE BASE DE DATOS REQUERIDA
---------------------------------
La tabla `uploads` debe tener al menos estas columnas:

  id         BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY
  user_id    BIGINT UNSIGNED NOT NULL  (FK → users.id)
  filename   VARCHAR(255)   NOT NULL   (nombre original)
  filepath   VARCHAR(500)   NOT NULL   (ruta relativa desde src/)
  mime       VARCHAR(127)   NOT NULL
  size       BIGINT UNSIGNED NOT NULL  (bytes)
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP


CONSIDERACIONES DE SEGURIDAD
------------------------------
- La extensión se sanea eliminando todo carácter no alfanumérico.
- El nombre generado no incluye el nombre original del usuario.
- Se usa is_uploaded_file() para prevenir ataques de file inclusion.
- El MIME se verifica con finfo (firma del archivo) en lugar de confiar solo en el cliente.
- Los mensajes de error de PDO se envían a error_log, nunca al navegador.
- Todos los valores mostrados en HTML pasan por htmlspecialchars(ENT_QUOTES, 'UTF-8').


NOTAS PARA PRODUCCIÓN (Render.com u otro hosting)
---------------------------------------------------
- El directorio src/uploads/ no es persistente en plataformas con filesystem efímero.
  Considerar almacenamiento externo (Cloudflare R2, AWS S3, etc.) y modificar el
  paso 2.8 para subir el archivo a la nube en lugar de a disco local.
- Asegurarse de que la extensión PHP fileinfo esté habilitada (php.ini: extension=fileinfo).
- El límite upload_max_filesize y post_max_size del php.ini debe ser al menos 10M.
